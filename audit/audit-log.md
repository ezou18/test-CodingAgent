# Audit Log

## Log Format

| Date       | Author | Action      | Description          |
|------------|--------|-------------|----------------------|
| 2026-05-11 | ezou18 | repo-init      | Initialize repository |
| 2026-05-11 | ezou18 | add-docs       | Add getting started guide |
| 2026-05-11 | ezou18 | add-answers    | Add Q1 answer |
| 2026-05-11 | ezou18 | add-docs       | Add contributing guide |
| 2026-05-11 | ezou18 | add-docs       | Add changelog and link from README |
| 2026-05-11 | ezou18 | conflict-merge | Reconcile two README description variants |
| 2026-05-11 | ezou18 | add-answers    | Add Q2 answer on pull requests |

## Categories

- **repo-init**: Repository creation and setup
- **add-docs**: Documentation additions or updates
- **add-answers**: Answer file additions or updates
- **conflict-merge**: Resolution of a merge conflict on a shared file
- **config-change**: Configuration modifications
- **access-change**: Permission or access updates

## Guidelines

1. Log every significant change to the repository
2. Use the standard format above
3. Include a brief but clear description
4. Keep entries in chronological order

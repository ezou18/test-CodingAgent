# Q3: What is a git worktree and when is it useful?

## Question

What does `git worktree` do, and why would you use it instead of just
switching branches?

## Answer

`git worktree` lets you check out multiple branches of the same repository
into separate directories simultaneously. Each worktree has its own working
directory and `HEAD`, but they all share the same `.git` object database.

You'd use it instead of `git checkout` when:

- You need to **work on two branches in parallel** without stashing or losing
  uncommitted changes on the current branch.
- You want to **run a long task** (build, test) on one branch while editing
  on another.
- You're reviewing a PR locally while keeping your in-progress branch intact.

## Example

```bash
git worktree add ../wt-feature-a -b feature/a
git worktree add ../wt-feature-b -b feature/b
git worktree list
# /repo               abc123 [main]
# /repo/../wt-feature-a def456 [feature/a]
# /repo/../wt-feature-b ghi789 [feature/b]
```

When done, remove the worktree:

```bash
git worktree remove ../wt-feature-a
```

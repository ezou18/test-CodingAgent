# Q2: What is a pull request and why do we use one?

## Question

What is a GitHub pull request, and why is it preferred over pushing directly
to `main`?

## Answer

A pull request (PR) is a proposal to merge a set of commits from one branch
into another. On GitHub, opening a PR creates a review surface where the
diff, commits, CI status, and discussion all live in one place.

PRs are preferred over direct pushes because they:

- **Enable review.** Teammates can comment on specific lines before code lands.
- **Gate on CI.** Tests, linters, and required checks can block a merge.
- **Document intent.** The PR title and description explain *why* a change was
  made, beyond what `git log` shows.
- **Keep `main` shippable.** Work in progress lives on feature branches until
  it's ready.

## Key takeaways

- Branch → commit → push → open PR → review → merge.
- Prefer small, focused PRs over large ones — they're easier to review.
- Resolve conflicts on the feature branch, not on `main`.
